# Hotel Booking System - Spring Framework Final Project

## Описание проекта

Этот проект представляет собой бэкенд-составляющую сервиса бронирования отелей. Приложение позволяет пользователям бронировать отели на определённые даты, искать отели по различным критериям, а также управлять контентом через административную панель CMS. Администраторы могут выгружать статистику по работе сервиса в формате CSV-файла.

## Основные функциональные возможности
### Управление отелями
- Создание, редактирование, удаление и поиск отелей.
- Возможность фильтрации отелей по различным критериям (название, город, рейтинг и т.д.).
- Пагинация списка отелей.

### Управление комнатами
- Создание, редактирование, удаление и поиск комнат.
- Фильтрация комнат по цене, количеству гостей, датам заезда и выезда.
- Пагинация списка комнат.

### Управление пользователями
- Регистрация новых пользователей с ролями (пользователь или администратор).
- Поиск пользователей по имени.
- Аутентификация и авторизация с использованием Spring Security.

### Бронирование комнат
- Бронирование комнат на указанные даты.
- Получение списка всех оформленных бронирований.
- Проверка доступности комнат на выбранные даты.

### Рейтинг отелей
- Возможность пользователей выставлять оценки отелям (от 1 до 5).
- Автоматический пересчёт рейтинга отеля на основе новых оценок.

### Статистика
- Сбор статистики по регистрациям пользователей и бронированиям комнат.
- Выгрузка статистических данных в CSV-файл (доступно только администраторам).

## Документация API
Для удобства разработчиков и интеграции с другими системами, документация API доступна через OpenAPI (Swagger). Вы можете просмотреть все доступные методы, их параметры и ответы в интерактивном интерфейсе Swagger UI.

## Как запустить документацию
### Запустите приложение.

##### Перейдите по адресу: http://localhost:8080/swagger-ui.html
##### В интерфейсе Swagger вы увидите список всех доступных эндпоинтов, их описание и возможность тестирования.

## Примеры основных эндпоинтов
### Управление отелями
- GET /api/hotels - Получить список всех отелей с пагинацией и фильтрацией.
- GET /api/hotels/{id} - Получить информацию об отеле по ID.
- POST /api/hotels - Создать новый отель (доступно только администраторам).
- PUT /api/hotels/{id} - Редактировать информацию об отеле (доступно только администраторам).
- DELETE /api/hotels/{id} - Удалить отель (доступно только администраторам).

### Управление комнатами
- GET /api/rooms - Получить список всех комнат с пагинацией и фильтрацией.
- GET /api/rooms/{id} - Получить информацию о комнате по ID.
- POST /api/rooms - Создать новую комнату (доступно только администраторам).
- PUT /api/rooms/{id} - Редактировать информацию о комнате (доступно только администраторам).
- DELETE /api/rooms/{id} - Удалить комнату (доступно только администраторам).

### Управление пользователями
- POST /api/users/register - Зарегистрировать нового пользователя.
- GET /api/users/{username} - Получить информацию о пользователе по имени.
- POST /api/users - Создать нового пользователя (доступно только администраторам).

### Бронирование комнат
- POST /api/bookings - Забронировать комнату на указанные даты.
- GET /api/bookings - Получить список всех бронирований (доступно только администраторам).

### Рейтинг отелей
- POST /api/hotels/{id}/rate - Поставить оценку отелю (от 1 до 5).

### Статистика
- GET /api/statistics/export - Выгрузить статистику в CSV-файл (доступно только администраторам).


## Инструкция по запуску
Клонируйте репозиторий:

```
git clone https://github.com/gatalya01/Hotel-service.git

cd hotel-booking-system
```

Запустите контейнеры с помощью Docker Compose:

```
docker-compose up -d
```
Запустите Spring Boot приложение:
```
./mvnw spring-boot:run
```
После запуска приложение будет доступно по адресу:

```
http://localhost:8080
```

## Настройка Spring Security
Приложение использует Basic Auth для аутентификации и авторизации. Доступ к большинству эндпоинтов требует авторизации, за исключением регистрации новых пользователей.

#### Роли:

- USER - обычный пользователь, может бронировать комнаты и ставить оценки отелям.

- ADMIN - администратор, может управлять отелями, комнатами и пользователями, а также выгружать статистику.

## Сбор статистики
Сбор статистики реализован с использованием Kafka и MongoDB. При регистрации нового пользователя или бронировании комнаты, событие отправляется в Kafka, а затем сохраняется в MongoDB. Администраторы могут выгрузить статистику в CSV-файл через соответствующий эндпоинт.

### Рекомендации по разработке

##### Для написания бизнес-логики используйте Spring MVC и PostgreSQL.

##### Для сбора статистики используйте MongoDB.

##### Следуйте требованиям, описанным в каждом задании, чтобы обеспечить полное соответствие функциональности.
